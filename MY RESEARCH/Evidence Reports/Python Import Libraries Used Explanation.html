Core Networking & Web Libraries
requestsHTTP client · session mgmt

Primary HTTP library for making requests, managing persistent sessions (requests.Session()), handling redirects, SSL/TLS verification with certifi, and capturing response objects (status, headers, body). Used extensively for passive scanning and fetching endpoints.

urllib.parseURL parsing

Provides urlparse, parse_qs, urljoin, and unquote to break down URLs, extract query parameters, rebuild absolute paths, and decode percent-encoded characters. Essential for detecting sensitive data in URL parameters and analyzing redirects.

Data Processing & Pattern Matching
reregular expressions

Powers almost every detection module: credit card patterns, JWT token extraction, stack traces (Python/Java/PHP), database errors, GraphQL introspection, internal IP addresses, open redirect parameters, CSP weaknesses, and mixed content scanning.

jsonserialization · HAR parsing

Processes HAR (HTTP Archive) files, loads JSON configuration, analyzes GraphQL responses, decodes JWT payloads, and generates final security reports with all findings organized by severity and category.

xml.etree.ElementTreeXML parsing (Burp Suite)

Parses Burp Suite XML exports. Traverses the document tree to extract base64-encoded requests/responses, status codes, headers, and reconstructs HTTP transactions from penetration testing sessions.

Cryptography & Token Security
base64decoding

Decodes base64 content from JWT tokens (three base64url parts), Burp XML response data, and HAR file content. Reveals plaintext for security analysis such as sensitive claims or leaked credentials.

certifitrusted CA bundle

Provides Mozilla’s curated certificate authority bundle. The HTTP session uses it to validate SSL/TLS certificates, ensuring secure connections without trusting invalid or self-signed certs.

jwtPyJWT · optional

Optional but powerful: decodes JWT headers and payloads, verifies signatures, extracts expiration claims, detects algorithm confusion (e.g., 'none' alg) and missing exp claims. Falls back gracefully if absent.

✨ Optional import — if available, enables advanced JWT weakness detection.
HTTP/2 & Packet Analysis (advanced/optional)
h2 (hyper-h2)HTTP/2 protocol

Implements HTTP/2 framing and stream handling. Used when processing PCAP files: decodes connection preface, reassembles streams, extracts headers and response bodies from multiplexed HTTP/2 traffic. Optional, enabled via H2_AVAILABLE flag.

⚡ HTTP/2 support requires h2 library.
scapypacket manipulation

Powers PCAP processing: imports TCP, Raw, IP, and PcapReader. Reassembles TCP streams from packets, respects sequence numbers, extracts HTTP messages, and handles PCAP streaming to avoid memory overload. Optional but critical for network capture analysis.

📡 Requires scapy — if missing, PCAP mode is disabled.
jsbeautifierJS formatting

Optionally beautifies minified JavaScript before analysis, making dangerous patterns (eval(), innerHTML, postMessage wildcards) easier to detect. Increases accuracy of client-side security scans.

System, Concurrency & Utilities
argparseCLI interface

Parses command-line arguments: input file path, number of workers, rate limit, timeout, max PCAP size, passive scan rate, and verbosity. Provides user control over analysis parameters.

datetime / timezone / timedeltatimestamps & durations

Handles UTC timestamps for findings, JWT expiration checks, elapsed scan time, and caching staleness. timedelta calculates token validity and measures request latencies.

uuidunique identifiers

Generates per-finding UUIDs for JSON reports, enabling deduplication and traceability across different endpoints.

timesleep · rate limiting

Implements passive rate limiting (time.sleep), measures request intervals, tracks last-seen timestamps for TCP stream garbage collection, and profiles total execution duration.

collections.defaultdictauto-initialized dicts

Simplifies severity statistics, category counts, and aggregation maps without manual key checks. Makes summary generation concise and bug-free.

typingtype hints

Imports List, Dict, Set, Optional, Tuple, Any, Union for robust code documentation and static analysis. Used throughout dataclasses and function signatures.

enum.Enumseverity & confidence

Defines type-safe enumerations for Severity (Critical, High, Medium, Low, Info) and Confidence (High, Medium, Low), preventing string-based errors and enabling consistent sorting.

File I/O & Path Management
pathlib.Pathmodern filesystem

Object-oriented path handling: extracts file extensions, checks sizes (PCAP max MB limit), validates existence, and builds output report paths.

Logging, Garbage Collection & Thread Safety
loggingdiagnostics

Structured logging with levels (DEBUG, INFO, ERROR). Reports PCAP parsing steps, HTTP/2 errors, and verbose processing details when --verbose is enabled.

gcgarbage collection

Explicitly triggers garbage collection every N requests (default 500) to prevent memory bloat, especially crucial when scanning thousands of URLs or processing large PCAP streams.

threading.Lock / Semaphoreconcurrency primitives

Lock guarantees thread-safe console output in PrintManager. Semaphore coordinates passive rate limiting across worker threads, preventing burst overloading of target servers.

concurrent.futures (ThreadPoolExecutor, as_completed)worker pool

High-level thread pool management: submits analysis tasks for each URL or HTTP transaction, collects results as they complete, handles exceptions per task, and respects max_workers limit.

HTTP Utilities & Cookie Parsing
http.cookies.SimpleCookieRFC 6265 cookie parser

Robustly parses raw Set-Cookie headers to extract attributes: Secure, HttpOnly, SameSite, Domain, Path, Max-Age, and Expires. Handles malformed headers and edge cases.

Supporting Data Structures
dataclasses.dataclassclean data containers

Automatically generates __init__, __repr__, and comparison methods for SecurityFinding and ObserverConfig. Reduces boilerplate while keeping logic clear and maintainable.

🔎 Summary of detection capabilities powered by these libraries:

  • requests + certifi – passive HTTPS scanning, security header validation.
  • re + json + xml – Info leaks (stack traces, DB errors), GraphQL introspection, JWT extraction.
  • scapy + h2 – PCAP reassembly, HTTP smuggling detection, HTTP/2 analysis.
  • jwt + base64 – Token weaknesses, missing expiration, sensitive payload fields.
  • jsbeautifier – JavaScript security (eval, innerHTML, postMessage wildcard).
  • concurrent.futures + threading – High-performance multi-endpoint analysis.

✅ All optional imports (JWT, h2, scapy, jsbeautifier) are wrapped with fallback logic, ensuring core functionality works even if advanced libraries are missing.
